# 允许只读联网查询设计

## 目标

允许 agent 在任务需要时自主进行只读联网查询，包括搜索公开信息和查阅官方文档，不再要求逐次获得用户批准。

## 权限边界

- 允许自主执行不会改变外部状态的公开信息查询。
- 涉及第三方库、框架、SDK、CLI、云服务或 API 时，优先查询对应版本的最新官方文档。
- 连接外部环境进行调试仍需用户批准。
- 安装依赖、联网下载、发布、推送及其他会改变本地或外部状态的操作仍需用户批准。
- 查询过程中继续遵守敏感信息保护要求，不提交或泄露密钥、令牌、凭证及个人隐私。

## 修改范围

仅修改 `AGENTS.md`：

1. 在“安全与敏感信息”中，将禁止主动联网改为允许自主进行只读联网查询，并保留敏感信息和安全机制约束。
2. 在“依赖与第三方文档”中，明确第三方技术问题可自主联网查询最新官方文档。
3. 不修改“命令与权限”中对外部环境调试、安装、下载、发布和推送的授权要求。

README、安装脚本和项目模板不需要修改，因为规则源、安装方式和项目级覆盖关系均未变化。

## 验证

- 检查 `AGENTS.md` 的联网相关条款不存在相互矛盾。
- 运行 `git diff --check` 检查文档格式。
- 运行 `bash -n install.sh` 确认安装脚本语法未受影响。
- 使用隔离 `HOME` 执行安装冒烟测试，确认规则仍能接入默认目标。
- 合并到 `main` 后重复关键验证，再推送并执行本机默认安装。

## Git 与安装流程

在 `codex/allow-network-lookups` 分支提交并推送变更；验证后本地合并到 `main` 并推送。确认远端主分支更新后，删除本地和远端开发分支。最后运行 `./install.sh`，默认接入 Codex 和 Claude Code。
