## 高风险变更清单

以下变更必须至少按中等任务处理；影响广或生产风险高时按重量任务处理，并说明影响范围、验证方式和回滚方案。按本项目实际删改：

- DB schema、SQL、索引和数据修复脚本。
- API 契约、DTO、枚举值、错误码和响应结构。
- 配置中心设置、环境变量和启动参数。
- MQ topic、消费者逻辑、重试机制和幂等逻辑。
- 定时任务、批处理任务和异步任务。
- 被多个服务使用的 Client / API 包或公共库。
- 金额、付款、税费、结算、白名单、权限、登录态、隐私和数据脱敏逻辑。
- 状态流转、审批流程、外部回调和第三方集成流程。

## SQL 与数据库规范（关系型数据库项目适用）

> 若本项目不使用关系型数据库，删除本节。

- 若项目有独立数据库规范文档，优先链接并遵循；否则以本节作为默认规范。
- 新增或修改建表 SQL 时，主键、时间字段、索引和表选项要显式、可读、可回滚。
- 涉及多表数据时优先评估查询规模、分页边界、空结果、`IN` 参数规模和索引命中情况。
- 不为图省事放宽查询条件或引入不可控全表扫描。
- 模糊查询默认优先左匹配；确需全模糊时说明原因、性能影响和验证方式。
- 项目如禁止联表查询，则必须拆成多个单表查询，并在应用内用 `Map`、集合或分组结构关联。
- 完成后清理本次需求内不再使用的 SQL、迁移脚本和数据修复脚本；删除前通过搜索或静态分析确认无其他引用，不确定时保留并说明原因。
- SQL、schema 或迁移变更需同步更新对应的 SQL/迁移说明文档，并在交付说明中记录回滚方式。

## 消息队列与异步任务（使用 MQ / 调度的项目适用）

> 若本项目不涉及 MQ 或异步调度，删除本节。

- 修改 MQ topic、消费者逻辑、重试机制或幂等逻辑前，先确认对上下游与重复消费的影响。
- 消费逻辑默认保证幂等；新增或调整重试机制时说明重试边界、退避策略和死信处理。
- 定时任务、批处理任务的并发与重入边界要显式说明。

## 资金、权限与数据安全（涉及相关业务的项目适用）

> 若本项目不涉及资金 / 权限 / 隐私数据，删除本节。

- 金额、付款、税费、结算、白名单、权限、登录态、隐私和数据脱敏逻辑属高风险路径，任何变更不得仅以“小改动”为由跳过测试。
- 涉及金额计算时明确精度、舍入规则和币种处理。
- 涉及权限与登录态时不放宽校验、不跳过鉴权；涉及隐私数据时遵循脱敏与最小披露原则。
